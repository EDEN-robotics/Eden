Refresher

EDEN builds cognitive layers for robots — memory, personality, and decision-making software.

We want robots that learn from experience, remember context, and make informed decisions — not just execute hard-coded commands.

Last Semester Recap

What we built in Fall 2025

5-Layer Architecture

Input, Context Gathering, Cognitive, Planning, and Action layers — end-to-end pipeline from perception to robot control.

Working Cognitive Layer

Short-term memory (Redis), long-term relational memory (Postgres + Vector DB), and personality module.

ROS MCP Action Layer

Validated in simulation — natural language to ROS2 commands via Model Context Protocol.

Key Decision

Pivoted from humanoid to simpler wheeled robots

Software is the core differentiator. Simpler hardware = faster iteration on the cognitive stack.

Lesson Learned

First version was buggy

Architecture was sound but not modular or production-ready. This semester: making every layer work properly.

What Didn't Work

ROS MCP + Cosmos reasoning had issues

Robot couldn't navigate properly. Poor spatial awareness. Needs RL training in simulation.

Layer 1

Input Layer
Improvements

Input Layer

Need a smaller, more efficient VLM. Input processing needs optimization for real-time on-device inference.

VLM Replacement Research

Evaluate MobileVLM, nanoLLaVA, TinyLLaVA. Benchmark inference time and accuracy on Jetson. Target: <200ms per frame.

Jetson Deployment

Set up Jetson in lab running continuously for data collection. Configure automatic restart and error recovery. Target: 90%+ uptime.

Frame Storage System

Purchase SSD for important frame storage. Implement frame importance scoring algorithm. Store high-importance frames locally before SuperMemory integration.

Layer 2

Context Gathering
Upgrades

Context Gathering

Context data sent to cognitive layer needs better filtering and relevance scoring.

Enhanced Context Processing

Multi-modal context fusion (visual + spatial + temporal). Add relevance scoring. Filter redundant/low-value context.

Environmental Understanding

Build spatial map of Turtle Lab. Track object locations and states. Scene understanding similar to NVIDIA/Tesla approaches.

Context Prioritization

Importance weighting system. Prioritize context relevant to current task. Summarize context for efficiency.

Layer 3 — Memory

Cognitive Layer:
Memory System

Cognitive Layer
SuperMemory Integration

Native package, knowledge graph visualization, optimized API calls, local caching layer.

Memory Decay & Consolidation

Short-term → Long-term pipeline. Forgetting curve for low-importance memories. Strengthen through repetition.

4 Types of Memory
Episodic — specific events
"Ian left a can on the table"
Semantic — facts & rules
"Lab rule: no food on tables"
Procedural — how-to knowledge
"How to clean up spills"
Social — relationships
"Ian is sometimes careless"
Importance Score Function
Importance = f( Relevance to function, Social relationship impact, Rule/safety implications, Frequency, Recency, Emotional valence )
Layer 3 — Personality

Cognitive Layer:
Personality & Social

Personality Development

Define core traits (helpful, careful, rule-conscious). Trait-based response modification. Consistency checking.

Relationship Tracking

Track interaction history per person. Build trust/reliability scores. Adjust behavior based on past interactions.

Physical Feasibility Checking

Validate if cognitive decisions are physically possible. Spatial reasoning about object affordances. Predict outcomes before execution.

New: Discord Bot Integration

  • Text-based conversations with team members
  • Log interactions and build relationships
  • Extract social dynamics and preferences
  • Test personality responses before hardware deployment
Layer 4

Planning Layer
Enhancements

Planning Layer

Cosmos reasoning not working properly. Need proper output format, accuracy, and multi-step task decomposition.

Cosmos Reasoning Fix

Debug current implementation. Fix input/output format. Test with simple scenarios first, then scale up.

Alternative LLM Integration

Try NVIDIA Nemotron (free API). Benchmark against Ollama models. Focus on agentic AI capabilities and reasoning quality.

Planning Algorithm Upgrade

Multi-step task decomposition. Error recovery and replanning. Validate plans against physical constraints.

Sim→Real
transfer learning
Isaac Sim Digital Twin

Create accurate 3D model of Turtle Lab. Import into NVIDIA Isaac Sim with all furniture, equipment, and boundaries. Match real-world dimensions.

Reinforcement Learning Training

Train navigation policies in Isaac Sim. Obstacle avoidance, path planning, tool usage and manipulation. Transfer to real robot.

Imitation Learning (IL)

Record expert demonstrations. Train on basic tasks (pick up, navigate). Combine with RL for robust behaviors.

ROS-MCP Improvements

Better environment mapping and localization. Dynamic obstacle avoidance. Test extensively in simulation before hardware.
